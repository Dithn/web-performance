Mission

The mission of the Web Performance Working Group is to provide methods to measure aspects of application performance of user agent features and APIs.

Specs

Navigation Timing - ${latest-status}
<p>This document provides an interface for web applications to access timing information related to navigation and elements.</p>
Page Visibility (Second Edition) - ${latest-status}
<p> This specification defines a means for site developers to programmatically determine the current visibility state of the page in order to develop power and CPU efficient web applications. </p>
Performance Timeline - ${latest-status}
<p> This specification defines an interface for web applications to access timing information related to navigation and elements. It is used by other specifications, like User Timing. </p>
Resource Priorities - ${latest-status}
<p>This specification defines a means for site developers to programmatically give the User Agent hints on the download priority of a resource. This will allow User Agents to more efficiently manage the order in which resources are downloaded.</p>
Timing control for script-based animations - ${latest-status}
<p> This document defines an API web page authors can use to write script-based animations where the user agent is in control of limiting the update rate of the animation. </p>
Frame Timing - ${latest-status}
<p>Frame Timing helps web developers measure the performance of their applications by giving them access to frame performance data to facilitate smoothness measurements.</p>
Resource Timing Level 1 - ${latest-status}
<p>This specification defines an interface for web applications to access timing information related to HTML elements.</p>
Beacon - ${latest-status}
<p>This specification defines an interoperable means for site developers to asynchronously transfer data from the user agent to a web server, with the user agent taking the responsibility to eventually send the data.</p>
Long Tasks API 1 - ${latest-status}
<p>This document defines an API that web page authors can use to detect presence of “long tasks” that monopolize the UI thread for extended periods of time and block other critical tasks from being executed - e.g. reacting to user input.</p>
Paint Timing 1 - ${latest-status}
<p>This document defines an API that can be used to capture a series of key moments (First Paint, First Contentful Paint) during pageload which developers care about.</p>
Cooperative Scheduling of Background Tasks - ${latest-status}
<p>The requestIdleCallback method is a more appropriate way for scheduling background tasks during times when the browser would otherwise be idle.</p>
Device Memory - ${latest-status}
<p>This document defines a HTTP Client Hint header to surface device capability for memory i.e. device RAM, in order to enable web apps to customize content depending on device memory constraints.</p>
Reporting API - ${latest-status}
<p>The reporting API provides a generic reporting framework which allows Web developers to associate a set of named reporting endpoints with an origin. Various platform features (like Content Security Policy, Network Error Reporting, and others) will use these endpoints to deliver feature-specific reports in a consistent manner.</p>
Network Error Logging - ${latest-status}
<p>Navigation Error Logging defines an API to store and retrieve error data related to the previous navigations of a document.</p>
User Timing Level 2 - ${latest-status}
<p>This specification defines an interface to help web developers measure the performance of their applications by giving them access to high precision timestamps.</p>
User Timing - ${latest-status}
<p> This specification defines an interface to help web developers measure the performance of their applications by giving them access to high precision timestamps. </p>
Preload - ${latest-status}
<p>This specification defines the <code>preload</code> relationship of the HTML Link Element (<code>&lt;link&gt;</code>).</p>
Timing Entry Names Registry - ${latest-status}
<p>This registry is intended to provide a central location for enumerating identified interface types of PerformanceEntry objects, which contain various data metrics for the the full lifecycle of a web application.</p>
Performance Timeline Level 2 - ${latest-status}
<p>This specification extends the High Resolution Time specification by providing methods to store and retrieve high resolution performance metric data.</p>
High Resolution Time - ${latest-status}
<p>This specification defines an interface that provides the current time in sub-millisecond resolution and such that it is not subject to system clock skew or adjustments.</p>
High Resolution Time Level 2 - ${latest-status}
<p>This specification defines a JavaScript interface that provides the current time in sub-millisecond resolution and such that it is not subject to system clock skew or adjustments.</p>
Server Timing - ${latest-status}
<p>Server Timing, part of the performance timeline metrics, provides API access to request-response cycle performance metrics communicated from the server to the user agent.</p>
Resource Hints - ${latest-status}
